# ToDone - 기능 명세 및 애플리케이션 정책 (Policy)

본 문서는 현재까지 개발 완료된 `ToDone` (Tauri 기반 macOS 메뉴바 할 일 관리 앱)의 주요 기능 목록과 그 목적, 기술적 기반, 그리고 최종 사용자 가이드(사용 설명서)를 정의합니다.

## 1. 애플리케이션 개요 및 특징 (Overview)
`ToDone`은 데스크톱 화면의 공간을 낭비하지 않는 형태의 빠르고 가벼운 Tray(메뉴바) 전용 할 일 관리 앱입니다. macOS 고유의 네이티브 감성(`Frosted Glass`)을 적용하여 세련된 UX를 제공하며, 복잡한 기능 대신 '오늘 할 일'과 '반복 업무' 관리에 초점을 맞췄습니다.

### 1.1 기술 스택
- **프론트엔드**: React(Vite), Tailwind CSS, Zustand, shadcn/ui
- **백엔드/코어**: Tauri v2, Rust
- **데이터베이스**: 로컬 SQLite (`tauri-plugin-sql`)

---

## 2. 주요 구현 기능 및 목적 (Features & Purposes)

### 2.1 메뉴바 트레이 모드 (Accessory Mode)
- **기능**: 일반 데스크톱 앱 창 대신, Mac 상단 메뉴바의 트레이 아이콘을 클릭해야만 우측 상단에 미니 사이즈(폭 800px, 높이 750px)로 팝업됩니다. 창 바깥을 클릭하면 자동으로 숨어버립니다(Blur Hide).
- **목적**: 작업 중 방해받지 않고 순간적으로 할 일을 체크/추가한 뒤 바로 본 업무로 돌아갈 수 있는 '생산성 허브' 역할을 수행하기 위함입니다. 하단 독(Dock)에 아이콘이 뜨지 않아 깔끔합니다.

### 2.2 전역 캘린더 스트립 (Global TaskTray)
- **기능**: 앱 최상단에 항상 가로로 스크롤 가능한 일주일치 달력(데이트 픽커) 바(Bar)가 노출됩니다.
- **목적**: 내가 지금 보고 있는 앱 화면의 모든 데이터(Daily 등)가 '어느 날짜' 기준인지를 직관적으로 제어하기 위해 만들었습니다. 상단 날짜를 변경하면 하위 데이터가 즉각 동기화됩니다.

### 2.3 영역별 맞춤 탭 뷰 (4-Tabs Navigation)
앱 하단의 탭을 통해 할 일을 성격에 맞게 분리해서 봅니다.
1. **Daily 탭**: 오늘(혹은 상단에서 선택한 특정일)까지 마감인 작업들을 모아보는 최우선 뷰.
2. **Weekly 탭**: 이번 주 월요일~일요일 사이에 걸친 할 일들을 완료/미완료로 분리해 보여줍니다. (각 항목 우측에 마감일 배지 노출)
3. **Recurring 탭**: 매일/매주 반복되는 루틴 업무들을 독립된 그룹으로 모아 보여주며, 이번 분기의 렌더링 상황을 표시합니다.
4. **All (Kanban) 탭**: 전체 마감일과 무관하게 모든 작업을 3단 상태 칸반(Todo, In-progress, Done)으로 나열하고 마우스로 자유롭게 옮길 수 있습니다.

### 2.4 작업 상세 모달 (Task Detail Modal)
- **기능**: 'Add New Task'를 누르거나 기존의 할 일 카드를 한 번 클릭하면 나타나는 반투명 팝업 창입니다. Title, Description(메모), Due Date(마감일), Status, Recurrence(반복 속성)를 기입/수정할 수 있습니다.
- **목적**: 메인 화면이 복잡해지는 것을 막고 좁은 트레이 앱 특성상 한 화면 내에서 깔끔하게 '생성, 수정, 삭제'를 통일성 있게 완수하기 위한 디자인입니다.

### 2.5 스마트 반복 로직 (Smart Recurring Pipeline)
- **기능**: 사용자가 모달에서 `Daily`, `Weekly`, `Monthly` 등의 반복을 설정한 카드를 **'완료(Done)'** 처리하면, 기존 카드는 완료된 채로 두고 **다음 주기에 해당하는 똑같은 새 카드를 자동으로 복제 생성**합니다.
- **백그라운드 스케줄러**: 유저가 클릭을 잊고 맥북을 며칠 방치하더라도, 트레이 아이콘을 클릭해 **앱 창에 포커스가 잡히는 순간(혹은 1시간 주기의 백그라운드 타이머에 의해) 과거에 누락된 사이클들을 역추적하여 현재 날짜까지 남몰래 모두 자동 동기화(Insert)**해 줍니다.
- **목적**: 유저가 매일 반복되는 작업을 수동으로 매번 '초기화'하거나 새로 타이핑할 필요 없이 항상 최신 스케줄러로 무인 동작되도록 하는 핵심 자동화 기능입니다.

## 3. 최종 사용자 가이드 (Usage Manual)

앱을 처음 설치하고 사용하는 방법에 대한 가이드라인 튜토리얼입니다.

### 📌 1단계: 앱 켜고 작업 추가하기
1. 화면 우측 상단의 메뉴바 아이콘을 클릭하여 앱을 엽니다.
2. 상단 캘린더 스트립에서 원하는 날짜(기본 '오늘')를 클릭합니다.
3. 탭 리스트 최하단의 `+ Add New Task` 글자(고스트 버튼)를 누릅니다.
4. 예쁜 프로스티드 팝업 모달이 뜨면 **제목**(필수)과 필요시 메모(Description)를 적습니다. 시작일/마감일을 캘린더에서 골라 넣을 수 있습니다.
5. 우측 하단의 `Save`를 누르면 리스트에 방금 작성한 작업 카드가 나타납니다.

### 📌 2단계: 카드 수정 및 삭제하기
- 제목이나 마감일, 메모 등을 수정하고 싶다면 **등록된 카드의 몸통(흰색 배경 영역)**을 스치듯 가볍게 클릭하세요. 
- 다시 똑같은 생성용 모달 창이 그대로 나타나며, 내용을 고쳐 다시 저장할 수 있습니다.
- 영구적으로 작업을 지우고 싶다면, 수정을 위해 모달을 연 뒤 왼쪽 밑에 숨어있는 빨간색 **Delete** 버튼을 누르시면 됩니다. (별도의 경고창 없이 신속하게 삭제됩니다)

### 📌 3단계: 나만의 루틴(반복 작업) 세팅하기
- 매일 영양제 먹기, 매주 보고서 쓰기 같은 루틴은 새 태스크를 만들 때 가장 아래의 `Recurrence(반복 속성)` 파라미터를 찾아 `Daily`나 `Weekly`로 맞춰주세요.
- **[중요 팁]** 반복 작업을 처음 만들 때에는 마감일(Due Date) 캘린더 아이콘을 눌러 **'해당 루틴의 첫 시작일'을 명시적으로 꼭 고르시면 좋습니다**. 만약 귀찮아서 비워두고 저장(Save)하면, 현재 앱 맨 위 달력에서 골라져 있는 날짜가 강제로 첫 마감일이 됩니다.
- 저장 후, 반복 작업은 해당 목록들 사이에 섞여 우측에 돋보이는 `DAILY` 전용 배지를 단 채로 나타납니다.
- 해당 작업의 왼쪽 체크박스를 마우스로 콕 눌러 완료(취소선) 처리해 보세요! 여러분이 수동으로 다음 스케줄러를 세팅할 필요 없이 즉시 내일(또는 다음 달) 기한의 새로운 할 일이 뿅하고 미리 복사되어 다가옵니다.

### 📌 4단계: 전용 탭 활용과 칸반 뷰(Kanban) 넘나들기
- **Recurring 탭 구경하기**: 일반 Daily 탭은 단지 '그 날 해야 하는 일들'만 모여 있습니다. 정규 루틴들만 깔끔하게 보고 싶다면 제일 하단 메뉴의 `Recurring` 탭에 진입해 주기별(매일/매주/매월)로 정리된 표를 구경해 보세요.
- **칸반에서 직관적으로 끌어당기기**: 하단 우측 탭의 `All`로 넘어가면 마법의 3단 보드가 나옵니다.
해야 할 일(Todo)에서 한창 진행 중인 일(In-progress) 칸으로 마우스를 이용해 카드를 잡아서 던져보세요(Drag & Drop). 여러 작업들의 현재 상태를 가장 거시적이고 효과적으로 배치할 수 있는 파워풀한 방법입니다!
