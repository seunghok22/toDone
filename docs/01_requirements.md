## 1. 프로젝트 개요 (Overview)

- **목표:** 주간 작업 기록 및 일일 할 일을 빠르게 추적하고 관리할 수 있는 개인용 생산성 데스크톱 앱 개발.
- **개발 원칙:** AI Agent(Antigravity)를 주도적으로 활용하여 개발을 진행한다.
- **플랫폼:** Mac 상단 메뉴바(Tray) 전용 앱(Run Cat 참조)을 타겟으로 하되, 추후 Windows 환경도 호환되도록 설계한다.

## 2. 주요 기능 및 UI/UX (Features & UI)

앱의 화면은 크게 최상단 날짜 컨트롤러와 4가지 하위 영역(Tab)으로 구성됩니다.

**[공통 상호작용]**
- 앱 최상단의 **전역 캘린더 스트립(TaskTray)**을 통해 원하는 기준 날짜를 클릭하면 이하 모든 뷰가 해당 날짜 기준으로 동기화됩니다.
- 어느 뷰에서든 작업을 클릭하면 **작업 상세 모달(Frosted Glass Poup)**이 열리며, 여기서 세부 내용(Description) 수정, 완료 일정, 반복 속성을 통일하여 제어합니다.

**① 일간 작업 영역 (Daily Tab)**
- 상단 캘린더 스트립에서 선택한 날짜에 마감해야 할 작업들을 리스트업합니다.

**② 주간 작업 영역 (Weekly Tab)**
- 선택한 날짜가 속한 주(월~일) 전체에 걸친 할 일들을 모아봅니다. (각 작업 우측에 시각적 마감일 배지 부착)

**③ 반복 작업 전용 영역 (Recurring Tab)**
- 매일/매주/매월 반복되는 루틴성 작업들만 독립시켜 보여줍니다. 완료 여부를 동기화하며 다음 주기를 준비합니다.

**④ 전체 작업 영역 (All Tasks & Board Tab)**
- 모든 작업을 3가지 상태(**작업 전 / 작업 중 / 완료**)로 나누어 칸반 보드 형태로 모아서 보여줍니다. (드래그 앤 드롭 지원)

## 3. 기술 스택 및 아키텍처 (Architecture & Tech Stack)

- **핵심 프레임워크:** **Tauri** (웹 기술로 가볍고 빠른 데스크톱 앱 빌드 / 불가능할 경우 대안 탐색).
- **아키텍처 (FE 중심 설계):**
    - 전통적인 백엔드(BE)와 데이터베이스(DB) 서버 구축을 지양하고 프론트엔드(FE) 위주로 구성한다.
    - 개인 데이터만 다루므로 로컬 스토리지(SQLite, 파일 저장 등)를 우선 활용한다.
- **성능 최적화:** 백그라운드(Idle) 상태에서 CPU 및 메모리 점유율이 거의 발생하지 않도록 최적화한다.

## 4. 확장성 및 데이터 정책 (Scalability & Data)

- **데이터 보존:** 앱 업데이트 시 기존에 저장된 사용자 데이터가 절대 유실되지 않도록 마이그레이션 정책을 유지한다.
- **미래 확장성:** 추후 다른 사용자와 항목을 공유할 수 있는 '공유 페이지' 기능 추가를 염두에 두고 데이터 구조를 설계한다.

## 5. DevOps 및 배포 파이프라인 (DevOps & Release)

- **초기 세팅 (최우선 작업):** 본격적인 코드 작성 전, Agent 세팅과 CI/CD 파이프라인 구축을 먼저 완료한다.
- **배포 방식:** 비용이 발생하는 애플 개발자 프로그램(Apple Developer) 등록을 피하기 위해, **GitHub Releases**를 통해 `.exe`, `.dmg`, `.app` 형태의 실행 파일로 배포한다.
